Overview

Here you will find reference documentation to help you start integrating Careem into your app or website as quickly as possible. The chart below explains the flow of API calls:

API Reference

(updated: February 16, 2016)

Method End point Description
GET /v1/products List of Careem product details
GET /v1/estimates/time Get time estimates
GET /v1/estimates/price Get price estimates
POST /v1/bookings Create a booking request
GET /v1/bookings/{booking_id} Get booking details
PUT /v1/bookings/{booking_id} Update a booking
DELETE /v1/bookings/{booking_id} Cancel a booking
GET /v1/bookings/{booking_id}/track Track a booking

Flow of APIs

Host Urls

Environment Host URL
Production http://qa-interface.careem-engineering.com

Example request

http://qa-interface.careem-engineering.com/v1/products? ...

Authorization

The authorization token can be found from the dashboard, to use the token in sandbox mode it should have a prefix of test- so for example, if the token is aabbcc123aabb then for sandbox mode it should be test-aabbcc123aabb.

http://qa-interface.careem-engineering.com/

Error

In case of any error (error codes and description described below in each of the APIs) the API response status will not be 2xx & the error json model will be as the following sample with different codes and message for each case. In any case you will always get one error at a time, there will never be (up until this point of time ) multiple errors returned in one response for the calls you make.

HTTP/1.1 400 Bad Request
{
    "code": "ET-0003"
    "message": "No captains found near given location"
}

Estimates : Trip fare, time & distance estimation API

  • compare changes to:
  • v1

Provides information about the trip itinerary you requested from a price,distance & time point of view . it also tells you if this trip will apply a surge ( in case of a now booking ) or what is the maximum surge cap that will apply at the time of the trip happening in case of a later booking.

http://qa-interface.careem-engineering.com/v1/estimates/price

Permission: none

Header

Field Type Description
Authorization String

Application token.

Parameter

Field Type Description
start_latitude Number

Latitude component start location.
start_longitude Number

Longitude component start location.
end_latitude Number

Latitude component of end location.
end_longitude Number

Longitude component end location.
booking_type String

Now or Later booking type. In case of LATER booking you have to send a pickup time (below) and it has to be 1 hour in advance or else an error will be returned.

Allowed values: "NOW", "LATER"

product_id Number

Unique identifier representing a specific product for a given latitude & longitude.
pickup_time Number

This field is a UNIXTIMESTAMP in milliseconds. Pickup time for later booking. This field is only mandatory when booking type is LATER.

Success 200

Field Type Description
duration Number

Estimated time for trip.
distance Number

Estimated distance for trip.
currency_code String

ISO 4217 currency.
estimate String

Estimated price range for trip.
low_estimate Number

Lower bound of the estimated price.
high_estimate Number

Upper bound of the estimated price.
metric String

Metric can be Km or mile.
surge_model Object

Surge model that contains surge details. This model could be NULL in case no surge really apply.
  token String

Surge token, you will need this once you request a booking using our new public api for booking creation.
  surge_multiplier Number

If booking type is NOW The multiplier that will apply on top of the estimated price provided If booking type is LATER The maximum multiplier that might apply at the time when the booking is taking place.
  expiry_in_minutes Number

Expiry of surge, always in minutes, you should not send an expired token when requesting a ride or else the api will respond with Error so you refresh your value and send it again. This is important to avoid a case where customer gets an unexpected surge.
  • Success-Response: 
HTTP/1.1 200 OK
{
     "duration":251,
     "distance":2.838,
     "estimate":"AED 50.4 - 55.0",
     "surge":
     {
         "token":"v7n6sqn04d51aj3ottuj4kdblp",
         "multiplier":1.2,
         "expiry_in_minutes":1
     },
     "metric":"KM",
     "currency_code":"AED",
     "low_estimate":50.4,
     "high_estimate":55.0
 }

Reference

Below is a reference to all the enumerated types you will encounter for making requests and getting responses.

Booking Status

Status Description
UPCOMING Booking has been scheduled. No driver has been assigned.
DRIVER_ASSIGNED Driver has been assigned to the booking.
DRIVER_COMING Driver is on the way.
DRIVER_HERE Driver has arrived at the pick-up location.
TRIP_STARTED Trip has been started.
TRIP_ENDED Trip has been ended.
CANCELED Booking has been canceled.
REJECTED_DUE_TO_CAPACITY Booking was rejected because the car capacity for the said time has been reached.
REJECTED_DUE_TO_BLACK_OUT Booking was rejected because the said date was marked as a black-out date.
REJECTED_DUE_TO_NO_DRIVERS Booking was rejected because no drivers were found. This is primarly for NOW bookings.

Booking Type

Type Description
NOW Ride will be booked for now. Current time will be set as Pickup time. If driver will be available in nearby location then it will be assigned to booking.
LATER Ride will be booked for later time. User can select date and time for which s/he wants to book a ride.
